What is an Application Programming Interface?

There are several ways information within the database can be accessed. One accepted standard is through an Application Programming Interface, or API. An API is a structured way to allow for programmatic access of records from a database. The API is written to extract data from the database and transform it into a common format, such as JSON, in a process called serialization. For our API, we wrote the serializers within the Django REST Framework for every database table or custom query. We also integrated django-allauth and Dj-Rest-Auth for the generation of tokens for each user. Using tokens for authentication allows us to ensure data access is secure and that transactions by a user are tracked. There are several libraries in all programming languages that are specifically tailored toward obtaining and formatting data from an API.

All API documentation are automatically generated with the drf-yasg library. The drf-yasg documentation includes all Create, Read, Update, and Delete (CRUD) operations.
